# BDパネリング 仕様書 (Draft)

## 1. 概要 (Overview)
- **アプリケーションの目的**: BD（バンド・デシネ）風の自作WEB漫画の脚本制作ツール。大まかなレイアウト構成と、セリフ・モノローグの配分を直感的に行い、最終的なイメージを画面上で確認しながら作業できる。
- **最終出力**: Obsidian Fountain対応のMarkdown形式ファイルとしてエクスポート。

## 2. 解決したい課題・現在の不満点
- 脚本エディタとコマ内のテキストがリアルタイムで連動していない。
- コマの編集が直感的ではない。
- 編集時に複雑な記述（Fountain文法など）を意識させず、WYSIWYG（見たままが得られる）な操作感にしたい。
- プレビュー時のフォントサイズや見え方が実際の漫画のスケール感と異なる。

## 3. コア機能 (Core Features)
- **直感的なコマ編集 (WYSIWYG)**:
  - コマをクリックするとシームレスに入力モード（テキストエディタ）に移行。
  - 編集終了（フォーカスが外れる等）で、実際の漫画と同等のフォントサイズでの表示モード（プレビュー）に戻る。
- **テキストの自動判別（プレフィックス記法）**:
  - `*` : モノローグ
  - `>` または `＞` : 主人公のセリフ
  - `@` または `＠` : その他の人物のセリフ
  - 記号なし (ベタ打ち) : 状況説明（ト書き/アクション）
- **グリッドとコマの結合**:
  - 基本となるグリッド（3x3, 4x3, 3x4, 4x4など）を選択可能。
  - 基本グリッド上で隣接するコマを結合し、任意のサイズのコマを作成可能。複雑な手動リサイズは避け、結合による調整にとどめる。
- **単一ファイル（Fountain Markdown）の内部管理**:
  - 全てのデータは内部的に1つのFountain文法のMarkdownとして管理・シームレスに更新される。
  - サイドバーの「脚本エディタ」は、このFountain Markdownの**リアルタイムプレビュー用**として機能する。
- **Obsidian Fountain向けエクスポート**:
  - ページ区切り: `---`
  - コマ区切り: `### Panel N`（仮：Markdownの見出しレベル3などを想定）
  - セリフ/モノローグ表現: アプリ側でプレフィックスから標準的なFountain文法（キャラクター名+セリフ、ト書きなど）へ自動変換して出力。
- **セーブ＆ロード（ハイブリッド方式）**:
  - ブラウザの内部ストレージ（localStorage等）による自動保存。
  - 作業用の専用ファイル（.jsonなど）としてのエクスポートおよびインポート機能。

## 4. UI/UXの要件
- 最小限の表現で最終イメージが掴めるUI（Excalidraw風の集中できるモダンUIをベースとする）。
- 「ややこしい記述をしない」「見やすい」「すぐリザルトを確認できる」の3点を重視。
- **ページ管理とナビゲーション**:
  - 複数ページのミニプレビュー（サムネイル）を一覧できるUIを用意。
  - 作業への集中を妨げないよう、普段は隠しておくか半透明化し、必要な時だけ意識させる控えめなデザインにする。
